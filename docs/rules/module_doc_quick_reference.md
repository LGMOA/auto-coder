## 编写提示

### 💡 内容提示
1. **概述要精准**：一句话说清楚模块的核心价值
2. **示例要完整**：确保代码可以直接运行
3. **结构要清晰**：按照用户使用的逻辑顺序组织内容
4. **图表要准确**：Mermaid图要反映真实的依赖关系

### 🎯 质量标准
- **可用性**：新手能快速上手

- **准确性**：代码和说明保持同步
- **一致性**：与项目其他文档风格统一
